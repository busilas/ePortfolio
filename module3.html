  01 UNIT

Introduction to Secure Software Development

Required Reading & Reflection

During Unit 1, I delved into key principles for creating secure software, analyzing the shortcomings of conventional development methods like Waterfall and the advantages of contemporary approaches such as Agile. The flexibility and iterative nature of Agile make it the preferred choice for secure, dynamic software environments while promoting a collaborative and responsive development process.
I was also introduced to the Unified Modeling Language (UML), an effective tool for designing software systems using standardized diagrams. As part of our group work, I utilized Umbrello, an open-source UML tool, to create a flowchart illustrating steps that lead to a common security vulnerability identified by OWASP. This exercise demonstrated how UML can simplify complex design elements, aiding in the visualization of system interactions and ensuring accurate representation of secure software models.
Unit 1 also emphasized the importance of cultivating a risk-aware culture within organizations. I discovered that while developing secure software is crucial, it's equally vital to foster an environment where developers and stakeholders actively identify and address risks. Establishing this culture is fundamental for maintaining secure development practices throughout the software lifecycle.
Lastly, this unit included the formation of our project team. We set up a WhatsApp Group for daily communication, signed Team Contract, and created a shared Google Drive folder for collaborative work. This initial organization lays the groundwork for a successful team project by ensuring clear communication, shared responsibilities, and easily accessible resources.

  02 UNIT

UML Modelling to Support Secure System Planning

Required Reading & Reflection

In Unit 2, I developed skills in UML modeling to enhance secure system design within the Agile Software Development Life Cycle (SDLC). This unit emphasized the role of UML diagrams and flowcharts as essential tools for effective communication among stakeholders, beginning in the design phase and supporting development throughout the process.
A key focus of this unit was integrating security practices at each stage of the Agile SDLC. I explored best practices that proactively address security vulnerabilities by embedding protective measures in every phase of development. To build on this, I engaged in hands-on practice with UML diagrams that represent secure system structures and workflows. These included use case diagrams to map user interactions, sequence diagrams detailing step-by-step workflows, activity diagrams highlighting user actions, and class diagrams outlining the relationships, attributes, and methods involved in the system architecture. Each of these modeling techniques helped me visualize how security considerations influence the design and functionality of software systems.
To further understand the standards that guide secure system planning, I studied ISO/IEC guidelines on software security, with a particular focus on the ISO/IEC 27000 series. Familiarity with this industry-standard terminology and framework allowed me to apply consistent security principles in my UML models and to better understand the language that underpins secure software development.

This exercise highlighted specific security tasks relevant to each Scrum stage, emphasizing the importance of building robust, secure software from the outset. I also wrote a blog post on managing insider cybersecurity risks, drawing on five key terms from ISO/IEC 27000 Section 3. This activity reinforced the idea that managing human factors is crucial to achieving effective cybersecurity, illustrating that technology alone cannot create a secure environment.
