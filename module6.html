06 MODULE

Software Engineering Project Management.

In Software Engineering Project Management, the Project Manager acts as a conductor, blending various components to create a successful software product. Similar to how a conductor combines different musical instruments to produce a unified symphony, the Project Manager coordinates stakeholder requirements, technical teams, and project limitations while managing the "triple constraints" of schedule, budget, and quality.
This module has enhanced my comprehension of approaches like Behaviour-Driven Development (BDD) and various Software Development Life Cycle (SDLC) frameworks, each representing a unique musical genre that shapes project implementation. Through this coursework, I have acquired crucial theoretical knowledge and hands-on skills, equipping me to handle the intricacies of software development with innovation and accuracy, much like a conductor leading musicians towards a flawless rendition.

Main gols for the module:

1. Understand key principles of project management and their relationship with software engineering.

2. Explore advanced architecture, BDD, and SDLC models.

3. Integrate approaches to effectively manage conflicts and compromises.

4. Develop skills in analysis, design, software construction, and testing.

5. Address psychological and sociological aspects of human-computer interaction.

6. Reflect on personal development and anticipate future challenges in the field.

Introduction to Software Engineering Project Management

Required Reading & Reflection

Managing projects encompasses overseeing resources, calculating the necessary effort for software delivery, and strategizing design, deployment, and maintenance. Consequently, it is essential to possess a diverse skill set and emphasize implementation by utilizing project management methodologies such as Agile (including Scrum, Kanban, and Scrumban) to facilitate and expedite contemporary software development initiatives. These approaches are generally more effective than the waterfall model for extensive customer-oriented software creation and distribution because they concentrate on progressively improving the product through early user feedback. To grasp and evaluate the significance of project management, examples of unsuccessful software engineering projects were examined.
